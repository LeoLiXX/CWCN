# 安卓手机控制RIG收发CW的需求

可以装在安卓手机上的APK，支持通过蓝牙、OTG线连接手机，收发CW音频信号。或者直接通过VOX进行CW的发送(可选，如果可行性较低，建议放弃)。

核心功能：

* 自适应WPM和频率设置的CW Decode
* 通过音频或者CAT进行CW内容的发射

其他周边功能

* 易用性提升，支持常用的CW沟通句式，支持呼号配置，支持 RIG/ANT/QTH/Name/功率/梅登海格网格位置的配置，便于常见句式中引用
* 支持Repeat，间隔指定的时间重复，方便发起广泛CQ呼叫，期间如果接收到任何回复，中断发送循环
* 支持一些简单句式的自动回复循环，具体后面展开描述
* 方便的记录QSO，通联频率，通联时间UTC，支持QSO记录导出
* 智能根据上下文分析容错，
* 句式内容翻译成易懂内容，同时可以推荐下一句回复的句式。
* 支持一些特殊的配置，比如 9-->N 以及AS, SK, KN 等。
* 尽可能多的支持字符集，不支持的字符和标点，标红提示。
* 安全功能，SWR驻波过高时停止并禁止发射，测试发射状态等。

功能对标

* OTG线连接、蓝牙连接、VOX等，参见FT8CN（本地/FT8CN 文件夹下）
* CW Decoder 能力可以参考 CW Skimmer
* CW 发送能力可以参考CW Skimmer以及 Ham Radio Deluxe 自带的DM780工具包的CW功能分支

## 核心功能

### 自适应的CW解码

可以是一个比CW Skimmer更现代的开源版，除了支持贝叶斯统计算法等类似算法支持动态WPM外，考虑在三个维度超越

1. 鲁棒性（弱信号/强噪声）
2. 自适应能力（WPM + 操作员风格）
3. 可扩展性（SDR/AI/多信道）

以下是某个AI给的建议，可以讨论

#### 输入层

* 非连线/蓝牙连接模式下支持麦克风录音
* 连线/蓝牙模式下来自USB蓝牙录音设备
* SDR

#### 信号检测（关键差异）

* 是否考虑不使用传统的FFT阈值？
* 自适应频率锁定，包括自动寻找CW频率峰值以及支持表一
* 用类似与 Kalman Filter 跟踪频率，对于手抖、漂移，QSB有更稳定的表现
* 自动SQL阈值侦测

#### 特征提取（核心输入）

输出一个时间序列

```text
[
  (timestamp, amplitude, tone_on/off)
]
```

再转换成：

```text
[
  duration_1,
  duration_2,
  ...
]
```

#### 核心突破：贝叶斯 + HMM 解码

状态定义

```text
S = {dot, dash, intra_gap, char_gap, word_gap}
```

概率模型:

发射概率

P(duration∣state)

用高斯建模

dot ~ N(μ, σ)
dash ~ N(3μ, σ)

转移概率

P(st​∣st−1​)

dot → intra_gap（高概率）
dash → char_gap（中概率）

#### 解码算法

用：

Viterbi
或 Forward-Backward

#### 自适应的WPM (关键亮点)

不要直接算WPM

👉 用“隐变量估计”

实时更新：

```text
mu = 0.9 * mu + 0.1 * observed_duration
```

#### 多信道解码（SDR 杀手级功能）

输入瀑布图后

* 自动检测多CW信号

```text
FFT → 峰值检测 → 多通道 pipeline
```

#### UI 设计

* 瀑布图展示
* 实时解码文本
* WPM 变化曲线
* 信号质量指标评估（RST）

一些基本原则

* 正式的操作页面应当是全屏，顶部可以显示一些信息，当前推测的WPM，当前可能对话的呼号等。如果可能，估算一下当前接收到的信号的RST并显示在这里
* 实时解码情况的图表很重要，可以从DEBUG页面借鉴过来放在顶部
* 界面最大的区域是解码接收到的信息，标记当前这一段Sentence的时间，等。
* 底部有一个输入区域，支持快捷选择一些模板，比如 CQX2，CQX3，QRZ，等，这些模板支持定制，标题可以指定，模板内容自己定制。有一些占位符比如<MyCallSign> <TargetCallSign> <RIG> <ANT> <GRID> <Name> 等自由拼接。可以有一些默认预置，用户允许增加或者修改。用户也可以自己输入一些信息用于发送。
* 有一个当前噪声等级的指示，从某个位置戳开可以浮在当前界面的上方，允许调整SQL等级，确保低于当前噪声音量的信号被忽略，防止误解码
* 底部一排标准的功能按钮区域，可以放置一排按钮比如 通联，频谱，通联记录，设置 等。具体设计可以参考FT8CN，右侧也可以有一个可以拖拽的浮动SideButton区域。所有功能按钮Metro风格的图标加上少量文字辅助说明。
* 设置部分，参考FT8CN的布局风格。当前的DEBUG功能，可以隐藏到设置界面。所有跟设置相关的，拉平，分功能区域放在这里。

### 支持快捷语句的CW发送功能

这是一个可以在手机端完全取代CW Keying设备的功能，可以通过选择预置的CW内容模板，或者输入待发送的内容，完成CW呼叫。
同电台的互动方式可以是有线连接，蓝牙连接或者只是通过播放声音VOX方式完成TX。

有以下关键功能

* WPM可以很方便的在界面上拖动设置，实时调节发送内容的速度快慢。
* 支持Repeat的时间间隔，可以重复发送。
* 发送过程可以暂停，可以 Resume 或者 Rewind 到开头重来，界面上应当可以展示当前发送了哪些内容。
* 支持标准的Morse Code 码表的同时，支持一些特殊组合，比如KN，SK，AS 等
* 发送的同时，解码框能同时解码，这样的话，聊天内容能大体按照聊天的先后输出在解码框
* 我发射和接收到的信息分行显示，我能够区分发送和应答。
